## Base URL

```
https://csboard.com/v1
```

## API keys

Send your key as a Bearer token on **every** request:

```bash
curl https://csboard.com/v1/listings \
  -H "Authorization: Bearer csb_pub_..."
```

Keys are generated in your profile — [grab one here](https://csboard.com/profile?tab=api).
As a convenience you may also pass `?api_key=csb_pub_...` as a query parameter.

- Only [`GET /v1/health`](/api-reference/get-health) works without a key.
- Read keys can only read. **Buying** requires a separately enabled trading
  capability on the key — see [Buying](/buying).
- We store only `SHA256(key)`, never the raw key.

## Errors

Every error returns `{ code, detail }`:

```json
{ "code": "invalid_api_key", "detail": "Unknown or revoked API key." }
```

| Status | code | When |
| --- | --- | --- |
| 401 | `missing_api_key` | No key / wrong prefix |
| 401 | `invalid_api_key` | Unknown or revoked key |
| 401 | `too_many_failed_auth` | Too many bad keys from your IP (5-min ban) |
| 403 | `trading_not_enabled` | Key can read but not buy |
| 429 | `rate_limit_exceeded` | Over your limit — see [Rate limits](/rate-limits) |
