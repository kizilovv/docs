---
title: "Rate limits"
description: "Per-key limits and how to handle 429s."
---

Limits are **per key**.

| Limit | Scope |
| --- | --- |
| **30 / min** | Pointed reads — listings, prices, currency, balance, order status |
| **30 / min** | Buying — `POST /v1/orders` (spends money) |
| **1 / min** | Bulk snapshot — `GET /v1/prices/snapshot.ndjson.gz` |

Exceeding a limit returns `429` with `{ "code": "rate_limit_exceeded" }` and a
`Retry-After` header (seconds). The bulk snapshot returns
`{ "code": "snapshot_rate_limited" }`.

More than 30 bad keys per minute from one IP triggers a 5-minute ban.

Need more headroom? [Contact us](https://csboard.com) to raise your key's limit.
