Эндпоинт orders возвращает историю ваших покупок, сначала самые новые, с keyset-пагинацией. Фильтруйте по временному окну и по статусу, затем переходите к более старым страницам, передавая `meta.next_cursor` обратно как `cursor`. Каждый заказ содержит разбивку по предметам со статусом доставки.

**Требуется аутентификация.** Отправьте ключ как `Authorization: Bearer csb_pub_...`.

## Query-параметры

<ParamField query="limit" type="integer" default="50">
  Результатов на странице. Минимум `1`, максимум `100`.
</ParamField>

<ParamField query="start_unix_time" type="integer">
  Возвращать только заказы, созданные не раньше этого Unix-таймстампа (в секундах).
</ParamField>

<ParamField query="end_unix_time" type="integer">
  Возвращать только заказы, созданные не позже этого Unix-таймстампа (в секундах).
</ParamField>

<ParamField query="status" type="string">
  Фильтр по статусу заказа. Одно из: `completed`, `hold`, `delivering`, `pending`, `cancelled`, `failed`.
</ParamField>

<ParamField query="cursor" type="string">
  Keyset-курсор из `meta.next_cursor` предыдущего ответа. Передайте его, чтобы получить следующую страницу.
</ParamField>

## Поля ответа

<ResponseField name="data" type="Order[]" required>
  Массив заказов для этой страницы.

  <Expandable title="Order object">
    <ResponseField name="order_id" type="string" required>
      CSBoard order id.
    </ResponseField>
    <ResponseField name="steam_id" type="string | null">
      Steam ID, на который были доставлены предметы, если известен.
    </ResponseField>
    <ResponseField name="status" type="string" required>
      Одно из: `completed`, `hold`, `delivering`, `pending`, `cancelled`, `failed`.
    </ResponseField>
    <ResponseField name="custom_id" type="string | null">
      Ваш собственный id, если вы задали его при покупке.
    </ResponseField>
    <ResponseField name="currency" type="string" required>
      Всегда `"USD"`.
    </ResponseField>
    <ResponseField name="charged_total_usd" type="number" required>
      Итоговая сумма списания по заказу в USD.
    </ResponseField>
    <ResponseField name="item_count" type="integer" required>
      Количество предметов в заказе.
    </ResponseField>
    <ResponseField name="hold_until" type="datetime | null">
      Когда удерживаемые предметы заказа будут освобождены, либо `null`.
    </ResponseField>
    <ResponseField name="created_at" type="datetime" required>
      Временная метка создания в ISO 8601.
    </ResponseField>
    <ResponseField name="updated_at" type="datetime | null">
      Временная метка последнего обновления в ISO 8601, либо `null`.
    </ResponseField>
    <ResponseField name="items" type="OrderItem[]" required>
      Разбивка по предметам.

      <Expandable title="OrderItem object">
        <ResponseField name="market_hash_name" type="string | null" />
        <ResponseField name="price_usd" type="number | null" />
        <ResponseField name="status" type="string" required>
          Одно из: `hold`, `delivering`, `delivered`, `returned`.
        </ResponseField>
        <ResponseField name="tradable_at" type="datetime | null">
          Когда предмет выйдет из торгового холда Steam, либо `null`.
        </ResponseField>
        <ResponseField name="return_reason" type="string | null">
          Причина возврата предмета, если `status` равен `returned`. Одно из: `trade_timeout`, `declined`, `expired`, `rolled_back`, `unavailable`.
        </ResponseField>
        <ResponseField name="steam_trade_offer_id" type="string | null" />
        <ResponseField name="steam_trade_offer_finished_at" type="datetime | null" />
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="meta" type="object" required>
  <Expandable title="meta object">
    <ResponseField name="next_cursor" type="string | null" required>
      Передайте обратно как `cursor`, чтобы получить следующую страницу. `null` на последней странице.
    </ResponseField>
    <ResponseField name="per_page" type="integer" required>
      Размер страницы, использованный для этого ответа.
    </ResponseField>
  </Expandable>
</ResponseField>

## Пример запроса

```bash
curl "https://csboard.com/v1/orders?status=completed&limit=50" \
  -H "Authorization: Bearer csb_pub_..."
```

## Коды ошибок

| HTTP-статус | Код | Значение |
|---|---|---|
| 401 | `missing_api_key` / `invalid_api_key` | Отсутствует или некорректный API-ключ. |
| 422 | `invalid_request` | Некорректные query-параметры (например, `limit` вне допустимого диапазона). |
| 429 | `rate_limit_exceeded` | Слишком много запросов. Подождите время, указанное в заголовке `Retry-After`. |
