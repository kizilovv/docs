<Note>
  **Доступ по запросу.** Instant Sell включается на уровне аккаунта. `GET /v1/sell/status` доступен любому ключу, но котировки и продажа возвращают `selling_not_enabled`, пока доступ не выдан. Напишите в поддержку, чтобы запросить доступ.
</Note>

## Как это работает

1. **Котировка** — `POST /v1/sell/quotes` с трейд-ссылкой Steam возвращает все предметы этого инвентаря, которые мы готовы купить, с точной суммой в USD за каждый.
2. **Создание ордера** — `POST /v1/sell/orders` с выбранными `asset_ids`. Бот сети CSBoard отправляет трейд-оффер на трейд-ссылку продавца. Большие корзины могут разбиваться на до 3 офферов от разных ботов — массив `offers[]` отражает каждый.
3. **Продавец принимает** — ордер переходит в `received`. Идёт трейд-защита Steam (около 8 дней). Ожидаемая сумма видна в `GET /v1/balance` в поле `incoming_hold`.
4. **Расчёт** — после окончания холда ордер завершается, и вся сумма зачисляется на **ваш баланс CSBoard**. Других направлений выплаты нет: деньги всегда приходят на баланс ключа, создавшего ордер.

<Warning>
  Деньги зачисляются **только после расчёта**, никогда в момент принятия оффера. Статус `received` — это деньги в пути, а не на балансе. Планируйте кеш-флоу с учётом холда ~8 дней.
</Warning>

## Верификация офферов для ваших пользователей

Каждый оффер содержит профиль бота-отправителя и id трейд-оффера Steam:

```json
"bot": {
  "name": "CSBoard Bot #4",
  "avatar": "https://…",
  "steam_id": "7656119…",
  "profile_url": "https://steamcommunity.com/profiles/7656119…",
  "level": 30
},
"trade_offer_id": "7654321098"
```

Покажите эти данные в своём интерфейсе: продавец сверяет входящий оффер с ожидаемым `trade_offer_id` и `steam_id` бота до принятия — эта проверка закрывает скам с подменой оффера.

## Эндпоинты

Все эндпоинты живут под `/v1/sell` и используют тот же Bearer-ключ `csb_pub_`, что и остальной API: `GET /v1/sell/status` (доступен всем), `POST /v1/sell/quotes`, `POST /v1/sell/orders` (\+ заголовок `Idempotency-Key`, `external_id` — ваш идемпотентный id, `min_total_usd` — пол цены против дрифта), `GET /v1/sell/orders/:id` (и `?external_id=…`), `POST /v1/sell/orders/:id/cancel` (только до принятия оффера). Полные примеры запросов и ответов — в английской версии страницы: [Instant Sell (EN)](/guides/instant-sell).

## Жизненный цикл ордера

| Статус | Значение |
| --- | --- |
| `pending` | Ордер создан, офферы готовятся |
| `offer_sent` | Трейд-оффер(ы) доставлены продавцу |
| `received` | Продавец принял; идёт трейд-защита (~8 дней) |
| `completed` | Расчёт завершён — сумма на вашем балансе |
| `cancelled` | Отменён, либо оффер истёк / отклонён |
| `failed` | Не удалось выполнить — см. `fail_reason` |

## Лимиты

Котировки — 10 запросов/мин на ключ; ордера — 5/мин, 10 активных, дневной кап объёма \$2 000 по умолчанию. Лимиты поднимаются индивидуально — напишите в поддержку.
